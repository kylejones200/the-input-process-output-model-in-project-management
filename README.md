## Business context

Every project is has an input-process-output model. Given some input, we want to transform it into some output. Inputs can be resources, requirements, constraints, or assumptions. Processes refer to the work that is done to transform those inputs into outputs --- the actual project activities and techniques applied. And outputs are the deliverables, outcomes, and other results produced by the project.

Inputs, processes, and outputs are interconnected. The outputs from one phase become the inputs to the next (like a chain). For example, the initiation phase might produce a project charter as an output, and that charter then becomes an input to the planning phase. Getting a firm grasp on these relationships is crucial for project managers.
